# Incident Response Runbook

1. Confirm customer impact and affected tenant schemas.
2. Freeze risky jobs in BullMQ if payroll, notifications, or tenant provisioning are involved.
3. Preserve audit logs, application logs, traces, and database snapshots.
4. Communicate status through the incident channel and customer success owner.
5. Patch, roll forward, or restore according to the module runbook.
6. Complete post-incident review with corrective actions.
